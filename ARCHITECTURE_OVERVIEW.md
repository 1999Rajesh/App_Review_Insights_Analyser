## рҹҺҜ Key Architectural Decisions

### 1. Why FastAPI?
- вң… Async support for LLM calls
- вң… Auto-generated API docs
- вң… Type safety with Pydantic
- вң… High performance (Starlette based)

### 2. Why React + TypeScript?
- вң… Strong typing reduces bugs
- вң… Component reusability
- вң… Large ecosystem
- вң… Easy hiring (popular framework)

### 3. Why Groq?
- вң… 10-100x faster than alternatives
- вң… Free tier sufficient for MVP
- вң… Multiple model options
- вң… Simple API integration

### 4. Why In-Memory Storage (for now)?
- вң… Rapid prototyping
- вң… No database setup needed
- вң… Perfect for demos
- вҡ пёҸ Will migrate to PostgreSQL for production

### 5. Why Monolith Architecture?
- вң… Simpler deployment
- вң… Faster development
- вң… Easier debugging
- вң… Sufficient for current scale
- рҹ”® Can split to microservices later if needed
