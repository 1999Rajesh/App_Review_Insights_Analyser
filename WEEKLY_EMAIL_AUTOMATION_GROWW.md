**App:** Groww (in.groww)  
**Email:** Draft email with weekly pulse report

## 🎯 Overview

This guide shows you how to automatically generate and send a **Weekly Pulse Report** for Groww app reviews, containing:

- ✨ Top 5 themes from recent reviews
- 💬 Real user quotes
- 💡 Three actionable ideas
- 📊 Statistics summary

## 🚀 Step-by-Step Execution

### Option 1: Manual Trigger via UI

#### Step 1: Fetch Groww Reviews
1. Open the application: http://localhost:3001
2. Look for "🤖 Auto-Fetch from Google Play Store" section
3. Enter App ID: `in.groww`
4. Country: Select "India" (or keep US if India doesn't work)
5. Click "🚀 Fetch Play Store Reviews"
6. Wait 5-10 seconds

**Expected Result:**
```
✅ Successfully fetched 250 reviews from Google Play Store
```

### Option 3: API Call (Programmatic)

You can trigger the workflow programmatically:

#### Step 1: Fetch Reviews
```bash
curl -X POST http://localhost:8000/api/reviews/fetch-play-store \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "in.groww",
    "weeks": 8,
    "max_reviews": 500,
    "country": "us",
    "language": "en"
  }'
```

**Response:**
```json
{
  "message": "Successfully fetched 250 reviews from Google Play Store",
  "app_id": "in.groww",
  "fetched_count": 250,
  "total_in_database": 250
}
```

---

#### Step 2: Generate Report
```bash
curl -X POST http://localhost:8000/api/analysis/generate-weekly-report
```

**Response:**
```json
{
  "id": "report-uuid-123",
  "week_start": "2026-03-09T00:00:00",
  "week_end": "2026-03-15T23:59:59",
  "total_reviews": 250,
  "top_themes": [...],
  "generated_at": "2026-03-15T02:25:00Z"
}
```

---

#### Step 3: Send Email
```bash
curl -X POST http://localhost:8000/api/email/send-report \
  -H "Content-Type: application/json" \
  -d '{
    "recipient_email": "your.email@example.com"
  }'
```

## 🔧 Troubleshooting

### Issue 1: No Reviews Found

**Symptoms:**
```
No reviews found for app: in.groww
```

**Solutions:**

1. **Check App Availability:**
   - Verify app exists in selected country's Play Store
   - Try different country (US instead of India)

2. **Verify App ID:**
   ```
   Correct: in.groww
   Wrong: groww, Groww, IN.GROWW
   ```

3. **Test with Known App:**
   ```
   WhatsApp: com.whatsapp
   Instagram: com.instagram.android
   ```

---

### Issue 2: Email Not Sending

**Symptoms:**
```
Failed to send email: SMTP connection failed
```

**Solutions:**

1. **For Testing:** Use dummy credentials (current setup)
   ```python
   SENDER_EMAIL = "test@example.com"
   SENDER_PASSWORD = "dummy_password_for_testing"
   ```

2. **For Production:** Configure real Gmail
   ```python
   SENDER_EMAIL = "your.email@gmail.com"
   SENDER_PASSWORD = "your-app-password"  # App-specific password
   ```

3. **Enable Less Secure Apps (if using old Gmail):**
   - Go to Google Account Settings
   - Security → Less secure app access
   - Turn ON (not recommended for production)

4. **Use App Password (recommended):**
   - Enable 2FA on Gmail
   - Generate app-specific password
   - Use that in .env file

## 🎯 Best Practices

### 1. Timing

**Optimal Schedule:**
- **Frequency:** Once per week (Mondays at 3:35 PM IST)
- **Current (Testing):** Every 5 minutes
- **Recommended (Production):** Change to weekly

**How to Change to Weekly:**
```python
# In config.py
SCHEDULER_INTERVAL_MINUTES = 10080  # 7 days in minutes
```

Or use cron schedule:
```python
from apscheduler.triggers.cron import CronTrigger

trigger=CronTrigger(
    day_of_week='mon',
    hour=15,
    minute=35,
    timezone='Asia/Kolkata'
)
```

---

### 2. Review Volume

**Sweet Spot:** 200-500 reviews
- Too few (<100): Limited insights
- Too many (>1000): Slow processing, diminishing returns

**Adjust in Settings:**
```json
{
  "max_reviews_to_fetch": 500
}
```

---

### 3. Date Range

**Recommended:** 8 weeks (2 months)
- Recent enough to be relevant
- Long enough to spot trends
- Good balance for AI context

**Change in Settings:**
```json
{
  "review_weeks_range": 8
}
```

---

### 4. Email Content

**Keep It Concise:**
- One page maximum
- Bullet points over paragraphs
- Action items clearly highlighted
- Real quotes add authenticity

**Structure:**
1. Snapshot (stats)
2. Top themes (with sentiment)
3. Real quotes (verbatim)
4. Action ideas (prioritized)
5. Trending (momentum indicator)
